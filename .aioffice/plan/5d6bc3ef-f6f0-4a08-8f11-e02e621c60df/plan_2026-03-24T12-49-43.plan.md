# 修改登录接口支持 POST + JSON

## 任务概述
将登录接口从 GET 改为 POST，支持前端通过 application/json 请求体发送登录数据。

---

### 步骤 1: [x] 创建 LoginDTO 数据传输对象
**调查结果**：
- 当前 AuthController.login() 使用 @RequestParam 接收参数，只支持 GET 请求
- 需要创建 DTO 类来接收 JSON 格式的登录数据

**解决方案**：
- 已在 com.example.demo.dto 包下创建 LoginDTO 类
- 包含 account 和 password 字段（虽然当前代码只用到 account，但为未来扩展预留 password）

**涉及文件**：
- `src/main/java/com/example/demo/dto/LoginDTO.java`

### 步骤 2: [x] 修改 AuthController 登录接口
**调查结果**：
- 当前使用 @GetMapping("/login") 和 @RequestParam String account
- 需要改为 @PostMapping("/login") 和 @RequestBody LoginDTO loginDTO

**解决方案**：
- 将 @GetMapping 改为 @PostMapping
- 创建或复用 Result 统一返回类
- 使用 @RequestBody LoginDTO 接收 JSON 数据
- 保持现有的 JWT 生成逻辑
- 返回统一的 Result<String> 类型

**涉及文件**：
- `src/main/java/com/example/demo/controller/AuthController.java`

### 步骤 3: [x] 检查并创建统一的 Result 响应类
**调查结果**：
- 当前 login 方法返回 String 类型
- 用户提到使用 Result 类型进行统一响应

**解决方案**：
- 已创建一个通用的响应包装类 Result<T>
- 包含 code、message、data 等标准字段
- 提供了 success() 和 error() 静态工厂方法

**涉及文件**：
- `src/main/java/com/example/demo/common/Result.java`
