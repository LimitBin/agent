# 合并拦截器任务

## 任务大纲
- 修改 AuthInterceptor，整合租户识别逻辑
- 修改 WebConfig，移除 TenantWebInterceptor 注册
- 删除 TenantWebInterceptor 文件

### 步骤 1.1: [x] 修改 AuthInterceptor
**调查结果**：
- AuthInterceptor 已从 JWT 中提取 tenant_id 并通过 UserContext.setUser() 设置
- TenantWebInterceptor 的 afterCompletion 也清理 UserContext，但 AuthInterceptor 已有此功能
- 合并后只需保留 AuthInterceptor 的现有逻辑即可

**解决方案**：
- AuthInterceptor 已经包含了租户识别逻辑（从 JWT 提取 tenant_id）
- 无需额外修改 AuthInterceptor，只需移除 TenantWebInterceptor 的依赖

**涉及文件**：
- `config/AuthInterceptor.java`（无需修改）

### 步骤 1.2: [x] 修改 WebConfig
**调查结果**：
- WebConfig 同时注册了 AuthInterceptor 和 TenantWebInterceptor
- 两个拦截器都处理租户识别，存在冗余

**解决方案**：
- 移除 TenantWebInterceptor 的引用和注册
- 同时修复了 `/redis-test` 路径缺少前导斜杠的问题

**涉及文件**：
- `config/WebConfig.java`

### 步骤 1.3: [x] 删除 TenantWebInterceptor
**调查结果**：
- TenantWebInterceptor 文件位于 `config/TenantWebInterceptor.java`
- 其功能已完全被 AuthInterceptor 覆盖

**解决方案**：
- 删除 `TenantWebInterceptor.java` 文件

**涉及文件**：
- `config/TenantWebInterceptor.java`（已删除）
